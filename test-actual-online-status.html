Expected Result:

You should see recent location data with timestamps. Representatives with seconds_ago < 65 should be online.

-- Check latest location for each representative SELECT r.name, r.id, ll.timestamp, ll.latitude, ll.longitude, EXTRACT(EPOCH FROM (now() - ll.timestamp)) as seconds_ago, CASE WHEN ll.timestamp > now() - INTERVAL '65 seconds' THEN 'ONLINE' ELSE 'OFFLINE' END as status FROM representatives r LEFT JOIN LATERAL ( SELECT * FROM representative_live_locations WHERE representative_id = r.id ORDER BY timestamp DESC LIMIT 1 ) ll ON true ORDER BY r.name;

Debugging Steps:

  1. Check Console Logs: Open browser console to see detailed status information
  2. Verify Data: Make sure representative_live_locations table has recent data
  3. Check Timestamps: Ensure location timestamps are recent (within 65 seconds)
  4. Test Individual Reps: Use getRepresentativeStatusDetails(repId) to check specific representatives
  5. Monitor Changes: Watch console logs to see when status changes occur

⚠️ Common Issues:

  • No Location Data: If representative_live_locations table is empty, all reps will be offline
  • Old Timestamps: If location data is older than 65 seconds, reps will be offline
  • Wrong Representative IDs: Make sure representative_id in location table matches id in representatives table
  • Timezone Issues: Check if timestamps are in correct timezone

✅ Success Indicators:

  • Console shows detailed status for each representative
  • Online representatives have recent location data (< 65 seconds)
  • Offline representatives have old or no location data
  • Status changes are logged when they occur
  • Live map shows correct online/offline markers
